Cases for everykToTheNthChar:

  • n is less than 2
  • n is greater than or equal to 2
  • s is empty
    Rationale: There are two modes of behavior for this method. The first is returning the input String itself and the second is returning only certain characters found in the input String. The behavior is determined by the value of n. Additionally, the edge case of operating on an empty String should be handled properly.

Cases for containsSubSequence:

  • The sequence is present in the first part of the String
  • The sequence is present in the last part of the String
  • The sequence is present in the middle of the String (i.e. touching neither end)
  • The sequence and the String are equal
  • The sequence does not exist in the String
  • The sequence does not exist in the same order within the String
  • The sequence is longer than the String
  • The String is empty
  • The sequence is empty
    Rationale: The location of the sequence (which end it touches, if any) falls under the "first, last, middle" axiom; the search should correctly find the desired sequence regardless of its place within the String. Tests to do with the length of the sequence test potential boundary cases. Empty Strings prove a common edge case, and so both the sequence and the String are tested as empty.

Cases for subtract:

  • The beginning of the minuend String should be subtracted
  • The end of the minuend String should be subtracted
  • The middle (i.e. neither end) of the minuend String is subtracted
  • The subtrahend String is not contained in the minuend String
  • The minuend String and the subtrahend String are equal
  • The subtrahend String is longer than the minuend String
  • The minuend String is empty
  • The subtrahend String is empty
    Rationale: The location of the subtracted characters falls under the "first, last, middle" axiom; the subtraction should occur regardless of their place in the minuend String. A test of "no operation" should be performed in which no subtraction occurs. Tests to do with the length of either String test potential boundary cases. Empty Strings prove a command edge case, and so both the minuend and subtrahend Strings are tested as empty.

Cases for doubleValue:

  • An integer is converted
  • A negative integer is converted
  • A non-integer is converted
  • A negative non-integer is converted
  • The input String has a leading decimal point (e.g. .123)
  • The input String has a trailing decimal point (e.g. 123.)
  • The input String has a leading zero (e.g. 0123)
  • The input String has a trailing zero (e.g. 123.00)
  • The input String is empty
  • The input String is improperly formatted (e.g. 1.2.3)
  • The input String contains illegal characters (e.g. 1.0f)
    Rationale: Testing of this method can be categorized in one of two ways: numerical or formatting. There are four numerical types, identified by the presence or absence of a negative sign and a decimal point. All four should be tested. In the formatting category, there are legal edge cases and illegal ones. Legal cases include unnecessary zeroes as well as absent zeroes on either side of the decimal point. Illegal cases include an excess of otherwise valid characters (i.e. - and .) as well as altogether invalid characters. Illegal cases should be checked to throw the relevant exception.

Cases for everyNthCharOfWord:

  • n is less than or equal to the length of the longest word
  • n is greater than the length of the longest word
  • n is not a positive number
  • The input String is empty
    Rationale: The method will return only spaces when n is too great. Otherwise it will return a mixture of spaces and letters. If n is not positive, which doesn't make sense (what is the 0th or -1st letter in a word?), the method should return an empty String. As always, an empty String should be passed as input to test an edge case.

Cases for flipWords:

  • A single word is converted
  • Multiple words separated by a whitespace are converted
  • Multiple words separated by non-whitespace are converted
  • A word with leading non-whitespace is converted
  • A word with trailing non-whitespace is converted
  • An empty String is converted
    Rationale: Tests follow the "zero, one, many" axiom for input Strings with one word, multiple words, and (as always) an empty String. The method should also be tested to differentiate between whitespace, word characters, and non-word characters. The leading and trailing tests follow the "first, last, middle" axiom.
